Community and Support

  • Developers are encouraged to join our mailing list.
  • Users who do not wish to contribute directly to Gophercloud may give feedback by clicking on Issues and opening a bug or feature request ticket on Github.
  • For more personal interaction, we currently hang out on the #openstack-sdks channel on Freenode IRC (web access link).

Contributing to Gophercloud

Would you like to contribute a feature, API, or bug fix to Gophercloud?

  • Make sure a ticket for it exists on our Github issues page. Search existing issues to avoid duplicates.
  • It's not strictly required, but we really recommend discussing your concerns and design to solve the problem with a Gophercloud developer. Many issues concerning Gophercloud are known (particularly missing bindings!), or possible solutions might be better expressed using a different implementation or algorithm. This can avoid delays in code review later.
  • Fork the package and code, test, and document! Please try to follow general Go community guidelines when writing new code or maintaining existing code. Please try to document new functions, types, or packages. Be sure to adjust the godocs for any existing functions you've modified (if appropriate).
  • Submit a pull request! We'll review the code and give feedback. Make sure to include as many tests as appropriate. Where inappropriate, be sure to document why, somewhere, either in the code or in the git commit log.
  • And speaking of Git commit logs, be sure to reference the Github issues ticket in the git commit log message as well, as a courtesy to other developers. That will also help us to create changelogs in the future.

If you follow these steps, you should have no problems getting code committed into Gophercloud!
